复杂的问题
    1.如何实现订单的延迟关闭
        方案1：先存储到数据库，定时任务扫描(不适合数据量大的场景)
        方案2：通过mq死信队列实现
            使用死信队列：一个队列创建是可以绑定一个死信交换机，当队列中的消息过期不被消费时，将会被发送到死信交换机，
            死信交换机绑定的队列存储这些消息，消费者去消费死信队列中的这些消息。
            此方案的缺陷：不同时间延迟投递需要创建大量的死信交换机。            
        方案3：使用mq插件的方式：(rabbit-delayed-message-exchange)消息延迟投递插件
